Epic: Operational and Financial Reporting
Description: This epic provides management and staff with tools to analyze sales performance, 
merchant activity and stock turnover through on-screen visualization and printed reports.
User Story 1: Sales Turnover Reporting
- Title: As a Manager, I want to generate a turnover report for a specific period so that I can 
analyse company sales performance. 
- Acceptance Criteria:
o The system must allow users to select a specific "Start Date" and "End Date." 
o The report must display total quantities sold and the total revenue received. 
o The report must be viewable on-screen and formatted for printing. 
User Story 2: Merchant History Tracking
- Title: As a Manager, I want to generate an order history report for a specific merchant so that I 
can review their purchase/order history. 
- Acceptance Criteria:
o The report must include: Order ID, Order Date, Dispatch Date, Total Value and 
Payment Status (e.g Pending). 
o The system must include a "Totals" line at the end of the report to summarize the 
period.
User Story 3: Individual Merchant Activity Report
- Title: As a Manager, I want to generate a detailed activity report for a merchant so that I can 
review their ordering behavior. 
- Acceptance Criteria:
o The report header must display the merchant's contact details. 
o Each order in the list must show: individual items, quantities, item costs, total order 
cost and any applied discounts. 
User Story 4: Global Invoice Monitoring
- Title: As a Manager, I want to view all invoices raised against all merchants for a given period 
so that I can monitor company-wide billing activity. 
- Acceptance Criteria:
o The report must list every invoice issued within the user-defined date range across the 
entire system. 
o Each entry must include the Merchant Identifier, Invoice ID, Date, Amount, and 
Payment Status.
User Story 5: Stock Turnover Analysis
- Title: As a Manager, I want to generate a stock turnover report so that I can compare products
sold against products newly received. 
- Description: Requires accurate data from both IPOS-SA-ORD and IPOS-SA-CAT ensure these 
systems are operational before tackling this.
- Acceptance Criteria:
o The report must display quantities of products sold versus quantities of products
received into stock for the selected period. 
o Data must be accurately pulled from both the IPOS-SA-ORD (sales) and IPOS-SA-CAT 
(deliveries) records.
